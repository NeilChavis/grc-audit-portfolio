# GRC & Audit Portfolio

This repository includes real-world Governance, Risk, and Compliance (GRC) and IT Audit-focused projects that simulates carrying out security assessments and system evaluation.

Every project is based on a real-world business scenario where an information system is in use and needs to be formally evaluated using the NIST Risk Management Framework (RMF). These projects don't just work from theory; they also try to figure out how systems really work, find real risks, and write down security controls in a way that is in line with how things are done in the industry.

## Projects

### Project 1: Cloud File Sharing System (SaaS)
Under the guise of a consulting firm, this project mimics the initial security authorization process for a cloud-based file sharing system that deals with confidential company information and basic personally identifiable information. In order to back a risk-based permission decision, we need to evaluate the system's present security posture, find control gaps, and the system is already in use.

#### Key Deliverables:
- System Security Plan (SSP) aligned with RMF principles
- FIPS 199 system categorization (Moderate impact)
- NIST SP 800-53 control selection and implementation
- Security Assessment Report (SAR) with identified findings
- Risk Register documenting system-level risks
- Plan of Action & Milestones (POA&M) for remediation
- System Boundary Diagram (SaaS architecture and dependencies)
- Authorization Summary with risk-based recommendation

## Portfolio's Purpose
- Application of the NIST Risk Management Framework (RMF)
- Security control selection and real-world implementation mapping
- Evidence-based assessment and documentation practices
- Risk identification, analysis, and prioritization
- Understanding of SaaS environments and shared responsibility models
- Development of core GRC and IT audit artifacts used in system evaluation and authorization

## Notes
These projects are set up to show how GRC Analysts, ISSOs, and IT Auditors look at systems in real life. This consists of collecting data from stakeholders, examining system documentation, evaluating control implementation, and recording results to facilitate risk-informed decision-making.
More projects will build on this base by adding new sorts of systems and ways to test them.
