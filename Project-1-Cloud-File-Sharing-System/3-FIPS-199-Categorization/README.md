# FIPS 199 Categorization
The Cloud File Sharing System (CFSS) was put into groups according to FIPS 199 to find out how a security breach could affect the confidentiality, integrity, and availability. This classification shows what kind of data the system handles and how it helps Apex Consulting Group run its company.

## Security Categorization
| Security Objective | Impact Level |
| ------------------ | ------------ |
| Confidentiality    | Moderate     |
| Integrity          | Moderate     |
| Availability       | Moderate     |

SC CFSS = {(confidentiality, MODERATE), (integrity, MODERATE), (availability, MODERATE)}

## Justification
### Confidentiality: Moderate
Data breaches in the CFSS could have devastating consequences for the company. Personal information (PII), financial data, and data that is sensitive to the company are all stored in the system. Reputational harm, a decline in customer trust, and privacy issues could result from this information leaking out. The effect on business operations, however, will likely not be disastrous.
### Integrity: Moderate
The organization would be severely affected if the CFSS data were corrupted. The legitimacy, efficiency, and accuracy of the business might be compromised if client deliverables, internal documents, or financial records were falsified or altered. It is not believed that these consequences will significantly or catastrophically affect the organization's core operations.
### Availability: Moderate
Cloud File Sharing System (CFSS) downtime would have devastating consequences for the company. Delays in project execution and decreased operational efficiency would result from users not being able to access important files and cooperate efficiently. But there would be other ways for the group to keep running, so it wouldn't be a total disaster.
